Event-Driven Programming (EDP)

Событийно-управляемое программирование (СУП)

Событийно-управляемое программирование

парадигма, в которой выполнение программы определяется возникновением событий

Ключевые особенности:
  • Код не выполняется линейно, а ожидает события.
  • Есть механизм регистрации обработчиков событий.
  • Используется диспетчеризация событий (event dispatching) через цикл событий (event loop).

JavaScript


document.getElementById("btn").addEventListener("click", () => {
    console.log("Кнопка нажата!");
});

Python


import tkinter as tk

def on_button_click():
    print("Кнопка нажата!")

root = tk.Tk()
button = tk.Button(root, text="Нажми меня", command=on_button_click)
button.pack()

root.mainloop()

Component-Based Software Engineering (CBSE)

Компонентно-ориентированное программирование

методология, при которой система строится из независимых и повторно используемых компонентов

Основные принципы:
  • Модульность – каждая часть системы оформлена в виде независимого компонента.
  • Повторное использование – компоненты могут быть использованы в разных проектах.
  • Инкапсуляция – детали реализации скрыты, компоненты взаимодействуют через интерфейсы.
  • Связывание через события – компоненты реагируют на события и изменяют своё состояние

React (JS)


function ButtonComponent() {
    const handleClick = () => {
        console.log("Компонентная кнопка нажата!");
    };

    return <button onClick={handleClick}>Нажми меня</button>;
}
компонент можно многократно использовать в разных частях проекта

Tkinter (Python)


import tkinter as tk

class ButtonComponent:
    def __init__(self, root, text):
        self.button = tk.Button(root, text=text, command=self.on_click)
        self.button.pack()

    def on_click(self):
        print(f"Кнопка '{self.button.cget('text')}' нажата!")

root = tk.Tk()
button1 = ButtonComponent(root, "Кнопка 1")
button2 = ButtonComponent(root, "Кнопка 2")

root.mainloop()
класс ButtonComponent представляет собой переиспользуемый компонент, который можно создавать с разными параметрами

Основные понятия событийно-ориентированного программирования

Event

События

определенное изменение состояния системы или внешней среды, на которое можно реагировать

  • Клик мыши
  • Нажатие клавиши
  • Соединение с сервером
  • Получение данных от сенсора
  • Изменение значения переменной

Основные понятия событийно-ориентированного программирования

Subscription

Подписка на событие

процесс регистрации обработчика события, который будет выполнен при его возникновении

  • Назначать несколько обработчиков для одного события.
  • Разделять логику обработки.

Основные понятия событийно-ориентированного программирования

Emit

Генерация события

процесс отправки сигнала системе о том, что что-то произошло

  • В интерфейсах (пользователь нажал кнопку)
  • В сетевых приложениях (пришел новый WebSocket-сообщение)
  • В микросервисах (один сервис уведомляет другой)

Основные понятия событийно-ориентированного программирования

Event Loop

Цикл обработки событий

механизм, который постоянно ожидает новые события и вызывает соответствующие обработчики

Где используется:

  • В браузерах (JavaScript)
  • В асинхронных серверах (Node.js, asyncio)
  • В UI-фреймворках (Tkinter, PyQt)

Основные понятия событийно-ориентированного программирования

Event Queue

Очередь событий

структура данных, в которой события хранятся до обработки

Где используется:

  • В многопоточных системах
  • В браузерах
  • В серверных фреймворках

Основные понятия событийно-ориентированного программирования

Callback

Обратные вызовы

используются для выполнения кода после наступления события

React (JS)


function fetchData(callback) {
    setTimeout(() => {
        callback("Данные загружены");
    }, 2000);
}

fetchData((data) => {
    console.log(data);
});
callback() вызывается после завершения операции

Asyncio


import asyncio

async def fetch_data():
    await asyncio.sleep(2)
    return "Данные загружены"

async def main():
    data = await fetch_data()
    print(data)

asyncio.run(main())
await останавливает выполнение, пока данные не будут загружены

История развития и популярность подходов

  • Событийно-управляемое программирование развивалось вместе с GUI. Ранние графические интерфейсы, такие как X Window System (1984), использовали событийную модель. Сегодня она применяется в браузерах (JavaScript, React), серверных технологиях (Node.js, WebSockets) и GUI-фреймворках (Tkinter, PyQt, Electron).
  • Компонентный подход зародился с модульного программирования в 1960-х. В 1990-х появились COM, JavaBeans, а затем React, Vue, Tkinter и модульные микросервисные архитектуры.
